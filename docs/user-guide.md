## 1. ภาพรวมระบบ

**appQC — The Precision Ledger** คือระบบบันทึกผลการสุ่มตรวจคุณภาพสินค้า (QC Inspection) แบบ Web Application

### Role ในระบบ

| Role | ชื่อเรียก | สิทธิ์การใช้งาน |
|---|---|---|
| **operator** | พนักงานหน้างาน | บันทึก QC + ดูประวัติ |
| **qc_admin** | Admin ตรวจงาน | ทุกอย่างของ operator + จัดการ Supplier / รหัสของเสีย |
| **admin** | Admin System | ทุกอย่างของ qc_admin + จัดการ Users |

### หน้าจอในระบบ

| หน้าจอ | Operator | QC Admin | Admin |
|---|---|---|---|
| Login | ✅ | ✅ | ✅ |
| บันทึก QC | ✅ | ✅ | ✅ |
| ประวัติ | ✅ | ✅ | ✅ |
| Admin | ❌ | ✅ | ✅ |

## 2. การเข้าสู่ระบบ (ทุก Role)

1. เปิดเว็บ appQC — จะเห็นหน้า **Login**
2. กรอก **Email** ที่ Admin สร้างให้ (เช่น `sls02@cometsintertrade.com`)
3. กรอก **Password**
4. กดปุ่ม **"เข้าสู่ระบบ"**
5. หากข้อมูลถูกต้อง → เข้าหน้าบันทึก QC โดยอัตโนมัติ
6. หากข้อมูลไม่ถูกต้อง → แสดงข้อความแจ้งเตือนสีแดง

> **หมายเหตุ:**
> - บัญชีผู้ใช้สร้างได้โดย Admin เท่านั้น (ไม่มีหน้าสมัครเอง)
> - ทุกครั้งที่เปิดเว็บหรือปิดแท็บ ต้อง Login ใหม่เสมอ
> - กดปุ่ม **"ออก"** ที่มุมขวาบนเพื่อออกจากระบบ

## 3. คู่มือสำหรับ Operator (พนักงานหน้างาน)

Operator สามารถใช้งาน 2 หน้าจอ: **บันทึก QC** และ **ประวัติ**

### 3.1 บันทึก QC

เป็นหน้าหลักสำหรับบันทึกผลการสุ่มตรวจ

#### ขั้นตอนที่ 1: กรอกข้อมูลสินค้า

| ฟิลด์ | คำอธิบาย | ตัวอย่าง |
|---|---|---|
| วันที่ | วันที่ตรวจ (ค่าเริ่มต้น = วันนี้) | 2026-04-17 |
| SAP Code | รหัสสินค้าจาก SAP | `1110001` |
| รายละเอียดสินค้า | แสดงอัตโนมัติจาก SAP Code | beW,ALL DAY,FOUNDATION... |
| กลุ่มสินค้า | แสดงอัตโนมัติ | Foundation |
| Brand | แสดงอัตโนมัติ | beW |
| Sales | แสดงอัตโนมัติ | ชื่อพนักงานขาย |
| SCM | แสดงอัตโนมัติ | ชื่อ SCM |
| รหัส Sup SAP | รหัส Supplier จาก SAP | `10000138` |
| Sup Code | แสดงอัตโนมัติจากรหัส Sup SAP | A |
| Supplier | แสดงอัตโนมัติ | GUANGDONG EFA... |
| Lot No. | หมายเลข Lot | INV23-CWZ106 |
| จำนวนรับ | จำนวนที่รับทั้งหมด | 75600 |
| จำนวนตรวจสอบ * | จำนวนที่สุ่มตรวจ (จำเป็น) | 500 |

> **เมื่อกรอก SAP Code** → รายละเอียด, กลุ่มสินค้า, Brand, Sales, SCM จะขึ้นอัตโนมัติ
> **เมื่อกรอก รหัส Sup SAP** → Sup Code และชื่อ Supplier จะขึ้นอัตโนมัติ
> **เมื่อเปลี่ยน SAP Code / Sup SAP** → ข้อมูลจะ reset และดึงใหม่อัตโนมัติ

#### ขั้นตอนที่ 2: เพิ่มรายการของเสีย

1. พิมพ์ค้นหาในช่อง "ค้นหารหัสของเสีย หรือชื่ออาการ..."
   - ค้นได้ด้วย **รหัส** (เช่น `11001`) หรือ **ชื่ออาการ** (เช่น `จุดสี`)
2. **คลิกเลือกหลายอาการ** ที่ต้องการ — แต่ละอันจะติ๊ก ✅ (เลือกได้หลายอัน)
3. กดปุ่ม **"เพิ่มในรายการ"**
4. อาการที่เลือกจะรวมเป็น **1 แถว** ในลิสต์ด้านล่าง แสดงเป็น:
   - `11001, 11002` — รหัสรวม
   - `ไม่พิมพ์/Printing missing, พิมพ์ขาดหาย/Printing not complete` — อาการรวม
5. เลือก **ระดับความรุนแรง** (Critical / Major / Minor)
6. กรอก **จำนวน** ที่พบ
7. แนบ **รูปภาพ** ของเสีย ได้สูงสุด 3 รูป (กดไอคอนกล้อง)

> **ต้องการเพิ่มกลุ่มอื่น** ที่ระดับความรุนแรงต่างกัน → ค้นหาอาการอื่น → เลือก → กด "เพิ่มในรายการ" อีกครั้ง

#### ขั้นตอนที่ 3: ตรวจสอบ % ของเสีย

- **มุมขวาบน** จะแสดง **% ของเสีย** คำนวณอัตโนมัติ
- สูตร: `(จำนวนของเสียรวม / จำนวนตรวจสอบ) × 100`
- แสดงสีน้ำเงินหาก 0% / แสดงสีแดงหากมีของเสีย
- ด้านบนรายการของเสียจะแสดง: Critical: X, Major: X, Minor: X, รวม: X

#### ขั้นตอนที่ 4: บันทึก

1. กรอก **หมายเหตุ** (ถ้ามี)
2. กดปุ่ม **"บันทึก"**
3. รอสักครู่ (ระบบ upload รูปและบันทึกข้อมูล)
4. **Popup สรุปผล** จะขึ้นมา แสดงข้อมูลทั้งหมดที่กรอก (อ่านอย่างเดียว แก้ไขไม่ได้)
   - เลขที่ Order (เช่น QC26040001)
   - ข้อมูลสินค้า / Supplier
   - สรุปของเสีย + %
   - รายการ defect + รูปภาพ
5. กดปุ่ม **"ปิด"** → ฟอร์มจะ reset พร้อมกรอกรายการใหม่

#### ลบรายการของเสีย
- กดปุ่ม **"ลบ"** ที่แถวนั้น ๆ (ก่อนบันทึก)

#### ลบรูปภาพ
- ชี้เมาส์ที่รูป → กดปุ่ม **x** สีแดง

### 3.2 ประวัติ (History)

เป็นหน้าดูข้อมูลที่เคยบันทึกไปแล้ว (อ่านอย่างเดียว แก้ไขไม่ได้)

#### การใช้งาน

1. กดเมนู **"ประวัติ"** ที่แถบด้านบน
2. เห็นรายการ QC orders ทั้งหมด เรียงจากล่าสุด
3. **ค้นหา** ได้ตาม:
   - Order No. (เช่น QC26040001)
   - SAP Code
   - Brand
   - Supplier
4. **กดที่รายการ** → expand ดูรายละเอียดเพิ่มเติม:
   - ข้อมูลสินค้า / Supplier / จำนวน
   - รายการของเสียแต่ละอาการ + ระดับ + จำนวน
   - รูปภาพ (กดเพื่อดูขนาดเต็ม)
   - หมายเหตุ

#### ข้อมูลที่แสดง (สรุปต่อรายการ)

- เลขที่ Order / วันที่ / สถานะ
- SAP Code / รายละเอียดสินค้า
- Brand / Supplier / Lot
- **% ของเสีย** (แสดงตัวใหญ่มุมขวา)
- Critical / Major / Minor
- ตรวจ X / ดี X / เสีย X

## 4. คู่มือสำหรับ QC Admin (Admin ตรวจงาน)

QC Admin สามารถใช้งานทุกอย่างของ Operator ได้ **+ เพิ่มเติม**:
- จัดการ **Suppliers** (เพิ่ม / แก้ไข / ลบ)
- จัดการ **รหัสของเสีย** (เพิ่ม / แก้ไข / ลบ)
- ดูรายชื่อ Users (ดูอย่างเดียว)

### 4.1 หน้า Admin

กดเมนู **"Admin"** ที่แถบด้านบน (เฉพาะ qc_admin และ admin เห็น)

จะเห็น 3 แท็บ: **Suppliers** / **รหัสของเสีย** / **Users**

---

### 4.2 จัดการ Suppliers

#### ดูรายการ
- เห็นตาราง Supplier ทั้งหมด (Sup Code, SAP, ชื่อ, Category, Purchase)
- ค้นหาได้ตาม Sup Code / SAP Code / ชื่อ Supplier

#### เพิ่ม Supplier
1. กดปุ่ม **"+ เพิ่ม Supplier"**
2. กรอกข้อมูล:
   - **Sup Code** * (เช่น `A`, `1L`, `Pre-Pack`) — ห้ามซ้ำ
   - **SAP Code** (เช่น `10000138`)
   - **Supplier Name** * (ชื่อเต็มของ Supplier)
   - **Category** (เช่น PK MAKEUP)
   - **Status** (ACTIVE / INACTIVE)
   - **Purchase** (Import / Local)
3. กดปุ่ม **"บันทึก"**

#### แก้ไข Supplier
1. กดปุ่ม **"แก้ไข"** ที่แถวนั้น
2. แก้ไขข้อมูลในฟอร์ม
3. กดปุ่ม **"บันทึก"**

#### ลบ Supplier
1. กดปุ่ม **"ลบ"** ที่แถวนั้น
2. ยืนยันการลบ

---

### 4.3 จัดการรหัสของเสีย

#### ดูรายการ
- เห็นตาราง: Code / อาการ / Reason / Type
- ค้นหาได้ตามรหัสหรือชื่ออาการ

#### เพิ่มรหัสของเสีย
1. กดปุ่ม **"+ เพิ่มรหัสของเสีย"**
2. กรอกตามลำดับ:

| ฟิลด์ | คำอธิบาย | วิธีกรอก |
|---|---|---|
| **Type (แหล่งที่มา)** | ประเภทปัญหา | **Dropdown** — เลือกจากรายการหรือพิมพ์เพิ่ม |
| **Reason (จุดที่พบ)** | กลุ่มอาการ | **Dropdown** — เลือกจากรายการหรือพิมพ์เพิ่ม |
| **Running No.** * | รหัสของเสีย | พิมพ์ เช่น `11001` |
| **อาการ (Symptom)** * | คำอธิบายอาการ | เช่น `ไม่พิมพ์/Printing missing` |

**Dropdown มาตรฐาน Type:**
1. Defect/ข้อเสียหาย(ซ่อมไม่ได้)
2. Repair/ซ่อม
3. Short Shipment/ส่งของขาด
4. Scrap/ใช้ไม่ได้ (ทิ้ง)
5. Supplier/ผู้ผลิต
6. Customer/ลูกค้า

**Dropdown มาตรฐาน Reason:**
1. Logo/สิ่งพิมพ์
2. Appearance/ลักษณะที่ปรากฎ
3. Function/การใช้งาน
4. Component/ส่วนประกอบ
5. Bulk/ตัวยา
6. Machine/เครื่องจักร

3. กดปุ่ม **"บันทึก"**

#### แก้ไข / ลบ
- ทำแบบเดียวกับ Supplier

## 5. คู่มือสำหรับ Admin System

Admin System สามารถใช้งานทุกอย่างของ QC Admin ได้ **+ เพิ่มเติม**:
- สร้าง / แก้ไข / ลบ **Users**

### 5.1 จัดการ Users

อยู่ในหน้า **Admin → แท็บ Users**

#### เพิ่ม User
1. กดปุ่ม **"+ เพิ่ม User"**
2. กรอกข้อมูล:
   - **Email** * — อีเมลสำหรับ Login (เช่น `qc05@cometsintertrade.com`)
   - **Full Name** — ชื่อแสดงในระบบ
   - **Role** * — เลือกจาก Dropdown หรือพิมพ์ Role ใหม่:
     - `operator` — พนักงานหน้างาน (บันทึก QC + ดูประวัติ)
     - `qc_admin` — Admin ตรวจงาน (จัดการ Supplier / รหัสของเสีย)
     - `admin` — Admin System (จัดการ Users)
   - **Password** * — อย่างน้อย 6 ตัวอักษร
3. กดปุ่ม **"บันทึก"**

#### แก้ไข User
1. กดปุ่ม **"แก้ไข"** ที่แถวนั้น
2. แก้ไข Full Name / Role / Password (เว้นว่างเพื่อไม่เปลี่ยน Password)
3. กดปุ่ม **"บันทึก"**

#### ลบ User
1. กดปุ่ม **"ลบ"** ที่แถวนั้น (ลบตัวเองไม่ได้)
2. ยืนยันการลบ

> **ข้อควรระวัง:**
> - Role ที่พิมพ์เพิ่มเอง (ไม่ใช่ admin / qc_admin / operator) จะยังไม่มีสิทธิ์เข้าหน้า Admin — ต้องแจ้งผู้พัฒนาเพิ่ม policy
> - เฉพาะ **admin** เท่านั้นที่จัดการ Users ได้ (qc_admin ดูรายชื่อได้แต่เพิ่ม/แก้ไข/ลบไม่ได้)

## 6. สูตรคำนวณ

### % ของเสีย

```
% ของเสีย = (จำนวนของเสียรวม / จำนวนตรวจสอบ) × 100
```

- **จำนวนของเสียรวม** = Critical + Major + Minor (นับจากทุกรายการในฟอร์ม)
- **จำนวนตรวจสอบ** = ค่าที่กรอกในช่อง "จำนวนตรวจสอบ"
- คำนวณ **อัตโนมัติ** ทั้งบน UI และในฐานข้อมูล

### Order No.

- สร้างอัตโนมัติ: `QC` + `ปี(2หลัก)` + `เดือน(2หลัก)` + `ลำดับ(4หลัก)`
- ตัวอย่าง: `QC26040001` = ปี 2026 เดือน 04 ลำดับที่ 1

## 7. คำถามที่พบบ่อย (FAQ)

**Q: กรอก SAP Code แล้วไม่ขึ้นข้อมูล?**
A: SAP Code ต้องตรงแบบ exact (เช่น `1110001` ไม่ใช่ `111`) — ตรวจสอบรหัสให้ถูกต้อง

**Q: กรอก Sup SAP Code แล้ว Sup Code ไม่ขึ้น?**
A: ตรวจว่า Sup SAP Code ถูกต้อง (เช่น `10000138`) หากยังไม่มีในระบบ ให้แจ้ง QC Admin เพิ่มใน Admin → Suppliers

**Q: Login ไม่ได้ / ลืม Password?**
A: แจ้ง Admin System เพื่อ reset password ในหน้า Admin → Users → แก้ไข

**Q: อยากดูข้อมูลที่เคยบันทึกไป?**
A: กดเมนู "ประวัติ" → ค้นหาด้วย Order No, SAP Code, Brand หรือ Supplier

**Q: บันทึกแล้วแก้ไขข้อมูลได้ไหม?**
A: ไม่ได้ — ข้อมูลที่บันทึกแล้วเป็น read-only เพื่อความถูกต้อง หากต้องการแก้ไข แจ้ง Admin

**Q: รูปภาพของเสียอัพโหลดได้กี่รูป?**
A: สูงสุด 3 รูปต่อ 1 รายการของเสีย

**Q: เลือกรหัสของเสียหลายอันพร้อมกันได้ไหม?**
A: ได้ — ค้นหา → คลิกเลือกหลายอัน → กด "เพิ่มในรายการ" จะรวมเป็น 1 แถว
